공식문서 좋음

npm i vue-router 설치

라우터는 그냥 눈속임(키워드)

실제로는 페이지가 무조껀 하나인데 화면이나 주소를 바꿔주는것

vue에는 nuxt.js가 있음

        <!-- 바뀌는 화면이라 이름을 router-view로 설정 -->

안바뀌는 공통적인 부분을 레이아웃이라고 함

누를때 마다 컴포넌트들이 바뀌면서 주소가 바뀜

주소가 해시태그로 되어있는거 -> 해쉬 라우터(vue의 디폴트 라우터) -> 새로고침을 해도 동작하는 장점(중요!!!)


실무에서는 해쉬라우터를 거의 안씀(검색엔진에 안걸려서 안씀)
-> 히스토리 라우터를 씀
    mode: 'history',
-> #이 사라짐
-> 새로고침(서버에 전송하는 행위)하면 페이지가 안뜸;;
-> 라우터는 다 눈속임이라서 페이지가 안뜨는것(또한 서버는 모르는 상태)

즉, 히스토리 라우터를 쓰고 새로고침을 해결하기엔 서버가 필요하다!!

        { path: '/game/:name', component: GameMatcher } // 다이나믹 라우터 매칭   /game/number-baseball, /game/response-check 등..
>> 수많은 주소를 하나로 묶을 수 있음

GameMatcher.vue에서 조건에 따라 렌더링 가능

쿼리스트링을 사용할떄가 많음(실무에서)
>>대표적인 사용예
page=3&offset=10...
(보통 게시판에 많이 쓰임)

쿼리스트링은 서버에도 전달이됨

param이나 query를 많이씀

제로초 블로그 메뉴바 참조

category는 고정 뒤에 파람쓰만 달라짐
